# Siege of Life

## 项目概要

基于康威生命游戏的双阵营策略闯关游戏与关卡自定义模拟器，使用 Rust + Bevy 开发。

---

## 技术选型

| 层 | 方案 | 说明 |
|---|------|------|
| 游戏引擎 | **Bevy** | 主引擎，负责窗口、渲染、输入、摄像机、事件系统 |
| UI 框架 | **bevy_egui** | 主菜单、关卡列表、HUD、模拟器操作面板 |
| 序列化 | **serde + serde_json** | JSON 关卡数据读写、进度存档 |

---

## 主菜单

游戏启动后显示主菜单，包含两个入口：

- **开始游戏**：进入关卡列表，选择已解锁的关卡进行游玩。
- **自定义**：进入自定义模拟器，自由编辑布局与测试。

---

## 游戏模式

本游戏包含以下两种模式：

- **关卡模式**：按预设关卡顺序闯关，玩家在限定资源下摧毁高价值单位即可过关。
  - **关卡列表**：可选择已解锁的关卡进行游玩。
  - **解锁规则**：需通关前一关才能解锁下一关。
  - **进度保存**：自动记录已通关的关卡。
  - **关卡文件**：正式关卡的 JSON 数据内置于工程文件中，由开发人员添加。
- **自定义模拟器**：自由设置初始布局、部署资源等参数，用于测试策略或设计关卡。

> 网格尺寸固定为 60×40（宽×高，横向），所有模式通用。

---

### 模拟器可放置元素

- **基础细胞**：红方普通细胞、蓝方普通细胞
- **障碍与目标**：墙、炸弹、高价值单位
- **预设图案**：滑翔机、LWSS（轻型飞船），以及以下经典生命游戏结构：
  - **静物**：Block（方块）、Tub（菱形）、Boat（船形）、Loaf（面包形）、Beehive（蜂巢形）
  - **振荡器**：Blinker（闪烁器）、Toad（蟾蜍）、Beacon（灯塔）、Pulsar（脉冲星）

### 模拟器绘制方式

- **点击绘制**：点击单格放置当前选中的元素。
- **拖拽连续绘制**：按住拖拽连续放置，类似画笔。
- **预设图案放置**：选中图案后在画布上点击放置，或从元素面板直接拖拽到画布上放置。
- **擦除**：提供擦除模式，点击或拖拽以清除单元格。
- **撤回**：提供撤回按钮，撤销上一步绘制操作（类似 Ctrl+Z）。
- **清空全部**：一键清除网格上所有内容，操作前弹出确认提示。

### 模拟器辅助功能

- **网格线**：可切换显示/隐藏。

### 模拟器关卡数据管理

- **导出**：将当前布局保存为 JSON 格式的关卡文件。
- **导入**：导入已有的 JSON 关卡文件进行查看、测试或修改。

### 模拟器关卡预览测试

- **试玩模式**：进入实际关卡游玩体验，按关卡流程进行测试。
- **部署测试**：在预设的部署区域内拖拽放置滑翔机/LWSS（含旋转确认），验证部署范围与资源限制是否正确。
- **快速返回编辑**：试玩结束后可一键返回模拟器编辑状态，继续调整布局。

### 模拟器关卡参数配置

- **部署范围**：通过画笔绘制或拉取方框，标记红方允许部署的区域。
- **部署资源**：分别限定滑翔机和 LWSS 的可用部署次数。
- **胜利条件**：固定为所有高价值单位被全部摧毁。

### 模拟器信息显示

- **演化状态**：仅显示当前状态——运行中 / 已暂停。

### 模拟器演化控制

- **开始/暂停**：控制演化运行与暂停。
- **单步演化**：每点击一次往前演化一步。
- **速度调节**：直接输入数值（毫秒/步或帧/秒），调节演化速度。
- **重置**：一键恢复到编辑后的初始状态。
- **边界处理**：网格边界视为墙，细胞不会超出边界演化。

## 核心规则

- **双阵营**：红方（玩家）与蓝方（敌方），细胞演化遵循经典康威生命游戏规则（B3/S23）。
- **复活裁定**：死细胞复活时的阵营由周围多数邻居决定。红邻居多则复活为红，蓝邻居多则复活为蓝。（复活条件为恰好 3 个活邻居，3 不可平分，故不存在平局情况。）
- **规则适用范围**：B3/S23 演化规则仅对**普通细胞**生效。墙与高价值单位不参与演化，不受该规则影响。
- **非对称对抗**：玩家控制红方，蓝方由关卡预设。

---

## 单位类型

- **普通细胞**：红蓝双方的基础单位，完全遵循演化规则。
- **墙**：惰性障碍物。不可被覆盖、不参与演化、阻挡细胞移动。
- **特殊单位**：具有专属行为机制的半自主单位，不适用或部分适用标准演化规则。目前仅设计了**炸弹**：
  - 由关卡预设放置，非玩家部署单位。
  - 任何阵营的活细胞（红或蓝）与其相邻（触碰）即触发爆炸。
  - 被引爆时产生半径 5 的圆形爆炸范围，清除范围内所有单元格（无视阵营与类型）。
  - 爆炸会连锁引爆范围内的其他炸弹。
  - 炸弹本身不参与演化。
- **高价值单位**：蓝方保护的核心目标。不参与演化、不移动。被红方细胞触碰（相邻格）即判定摧毁。每个关卡可设置多个高价值单位，需**全部被摧毁**方可过关。

---

## 玩家操作与限制

- **部署区域**：红方可部署区域由关卡预设限定（不再固定为左半区）。
- **可用单位**：玩家可部署滑翔机和轻型飞船（LWSS），具体每关可用的单位类型与剩余次数由关卡预设限定。
- **部署方式**：从单位面板拖拽图标到画布上放置。
  - 拖拽时显示幽灵预览，合法位置为绿色/半透明，非法位置（部署区外、重叠等）预览变红。
  - 拖拽到合法位置后进入**旋转确认**阶段，玩家**同样通过鼠标拉拽旋转**选择单位朝向（上/下/左/右四个方向）。
  - 旋转确认后单位正式放置，触发放置特效并自动开始演化。
- **部署时机**：仅在演化暂停时可部署。
- **资源限制**：每关有固定的部署次数上限，分别显示滑翔机和 LWSS 的剩余次数。

---

## 流程结构

1. **部署阶段**：玩家在可部署区域内拖拽单位图标到画布上，先鼠标拖拽选择朝向（上/下/左/右）完成旋转确认，确认后单位正式放置（至少部署一个单位才能进入演化阶段）。
2. **演化阶段**：每次部署后自动连续演化解算 200 步，期间玩家不可操作。
3. **暂停与判定**：200 步演化结束后自动暂停，此时检查高价值单位是否**全部被摧毁**：
   - **全部摧毁** → 弹出胜利提示，回到关卡列表（下一关自动解锁）。
   - **未全部摧毁且仍有剩余部署次数** → 回到部署阶段。
   - **未全部摧毁且无剩余部署次数** → 弹出失败提示，提供"重试"和"回到关卡列表"选项。

## 视觉反馈要点

- 细胞生灭采用**闪烁/缩放**动画：诞生时闪一下从小变大，死亡时收缩消失。
- 单位放置、单位摧毁、高价值单位摧毁均配套动态特效（特效素材后续制作导入），其中高价值单位被摧毁时触发强调性特效，作为玩家成就感的核心反馈。
- 演化态与部署态需有清晰的视觉区分。
- 炸弹爆炸时触发**全屏震动**效果（覆盖整个窗口，不仅是画布区域）。
